# 多城市POI多样化与任务随机化改造计划

## 1. 数据库清理 (需确认)

* **目标**: 清除现有的密集北京 POI 数据和相关任务，为新数据腾出空间。

* **操作**:

  * `TRUNCATE TABLE poi;` (清空 POI 表)

  * `TRUNCATE TABLE task;` (清空任务表，因为任务依赖 POI)

  * `TRUNCATE TABLE task_route_point;` (清空任务路径点关联表)

  * **保留**: `driver`, `truck`, `customer` 表数据保留，因为它们是基础资源。

## 2. POI 数据初始化改造

* **目标**: 引入多个核心物流枢纽城市的 POI 数据。

* **实现**:

  * 修改 `PoiCrawler` 或创建一个新的初始化脚本。

  * 遍历城市列表：`["北京", "上海", "广州", "深圳", "成都", "武汉", "西安"]`。

  * 针对每个城市爬取少量关键 POI (如 "物流园区", "工业园", "商场")。

  * 控制每个城市的 POI 数量（例如每个城市爬取 1-2 页），避免数据过载但保证覆盖面。

## 3. 任务生成逻辑优化

* **目标**: 能够生成跨城市的随机任务，而不仅仅是北京内部或北京-上海。

* **实现**:

  * 修改 `TaskGenerator.generateRandomTasks`。

  * **起点选择**: 从所有 POI 中随机选择一个（不限城市）。

  * **终点选择**: 从所有 POI 中随机选择一个（不限城市），但确保与起点距离适中（例如 > 50km），或者完全随机。

  * 这样就会自然产生 "北京->广州", "武汉->成都", "上海->北京" 等多样化路线。

## 4. 执行验证

* 执行清理 SQL。

* 运行 POI 爬取逻辑初始化新数据。

* 生成新任务并验证起点/终点分布。
